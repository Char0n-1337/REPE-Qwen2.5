3、放开上下文限制观察越狱输出

4、单层（对20层）强度控制测试

5、对14 16 18 20层施加相同强度的测试结果，对上述四个层进行随机组合施加相同强度的控制向量。未完成：以及对上述层施加不同强度的测试结果

6、对14 16 18 20四层进行单层强度注入实验，发现单层低强度-1.5注入均无法实现越狱，四层-1.5注入实现越狱。未完成：双层和三层的随机组合实验，但代码逻辑出现错误，没有清空之前的注入效果，因此需要重新实验修正结果。

7、正向注入测试：结果发现正负向注入可以影响模型，但并非以0为对称点的镜像关系，负向可注入空间大比正向大

更新：基于REPE拓展论文发现恶意激活值在传播过程中呈现 上升 顶点 下降 的趋势，顶点为“恶意拦截层”,17层-4.0强度直接输出sql注入和XSS攻击等输出。21，22层(可能并不参与决策)无论如何注入都无法越狱，输出呈现强拒绝。

目前假设：Qwen2.5-3B-Instruct 的安全机制不是均匀分布的，而是主要集中在 18-20 层形成安全修正信号，在 21-22 层完成最终拒绝决策；23层则更接近输出重写层，容易改变最终表述但不一定能彻底推翻安全机制
